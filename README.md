How can I ensure that my code, program, or software is functional and secure?

I believe that the simplest answer to this question is testing! Effective and consistent testing will keep your code aligned with the requirements and functioning as intended. We have gone over many testing methods throughout this course and one of the main types that I utilized was unit testing. I utilized unit testing as I coded in order to make sure my code was up to par.

How do I interpret user needs and incorporate them into a program?

I think that you just need to make sure that your software adheres as closely as possible to all requirements set by the client. This involves testing against requirements rather than testing only what you have coded. Making sure to ask specific questions regarding requirements and how the client wants the final product to function will help to determine all of their needs and incorporate them into the software.

How do I approach designing software?

I always think of myself solving a complex problem when I design software. First I put together all of the individual blocks that I think would be involved. Then I have to determine how to incorporate all these pieces together. I try to keep testing each piece as I code so that I don't leave any major errors in the code. When I come up against problems I take a step back and think of solutions. So far these have worked well for me and I hope that these steps continue to help me as I finish this degree and begin working in this field.
